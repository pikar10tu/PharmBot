ใบเลือก Prompt & Config (สำหรับทีมตัดสินใจ)

เลือกชุดที่ดีที่สุด ณ ตอนนี้ เพื่อเดินหน้าทำ validation กับผู้เชี่ยวชาญให้ทัน

📅 25 มิ.ย. 2026 🟩 = อันแนะนำ ⏭️ การทดสอบ A/B = การศึกษาอนาคต
วิธีใช้: ทีมอ่านแต่ละตัวเลือก ดูข้อดี/ข้อเสีย แล้วทำเครื่องหมายในช่อง ☐ ที่เลือก · ช่อง “ชุดที่แนะนำ” ด้านล่างคือคำแนะนำของเรา ถ้าเห็นด้วยเลือกตามได้เลย

A เลือก Prompt ของผู้ป่วยจำลอง

ทั้งหมดอิงจาก buildSystemPrompt ที่มีอยู่ — ต่างกันที่ "แก้มาก/น้อย"

P1 — Prompt ปัจจุบัน (คงเดิม) ปลอดภัย

ใช้ 10 กฎ + voice overlay ที่มีอยู่ตอนนี้ ไม่แก้อะไรเลย

ข้อดีรู้พฤติกรรมแล้ว ไม่มีความเสี่ยงใหม่ เริ่ม validation ได้ทันที
ข้อเสียยังมีช่องโหว่: ถ้าเภสัชกรพูดอังกฤษ/ชื่อยาอังกฤษ ผู้ป่วยอาจหลุดไปตอบภาษาอังกฤษ (native audio สลับภาษาเอง)

P2 — ปัจจุบัน + ปักภาษาไทย ★ แนะนำ

เหมือน P1 ทุกอย่าง แต่เพิ่มกฎเดียว: “พูดภาษาไทยเสมอ ไม่ว่าเภสัชกรจะพูดภาษาอะไร แม้ถูกถามเป็นภาษาอังกฤษหรือมีชื่อยาภาษาอังกฤษ”

ข้อดีแก้ช่องโหว่เดียวที่ชัดเจน · เพิ่มแค่ 1 บรรทัด ความเสี่ยงต่ำมาก · คงพฤติกรรมเดิมที่ทำงานดีอยู่แล้ว · เริ่ม validation ได้เลย
ข้อเสียไม่ได้ปรับเรื่องความยาว/ความเป็นธรรมชาติ (แต่ของเดิมก็โอเคแล้ว)

P3 — Voice variant กระชับ + ปักภาษาไทย เก็บไว้ศึกษาอนาคต

ตัดกฎที่ไม่จำเป็นต่อการพูดออก เหลือเฉพาะกฎพฤติกรรมสำคัญ + ปักภาษา เพื่อลด over-instruction

ข้อดีอาจพูดเป็นธรรมชาติขึ้น
ข้อเสียเป็นการเปลี่ยนพฤติกรรมที่ยังไม่ทดสอบ → เสี่ยงได้ผลไม่คาดคิด เหมาะกับเฟสทดสอบ A/B ในอนาคต ไม่เหมาะรีบใช้ตอนนี้

✓ ชุดที่แนะนำ (Recommended Bundle)

สมดุลระหว่าง "ดีพอใช้งานจริง + ความเสี่ยงต่ำ + เริ่ม validation ได้ทันที"

Prompt: P2 (ปัจจุบัน + ปักภาษาไทย)
Model: gemini-3.1-flash-live-preview
เสียง: Aoede (เคสหญิง) / Puck (เคสชาย) — ตามเพศเคส
thinkingLevel: minimal  ·  VAD: LOW + prefixPadding 300ms  ·  temperature: 0.8
เหตุผล: เปลี่ยนจากของเดิมแค่ "ปักภาษาไทย" จุดเดียว ซึ่งเป็นการแก้ที่ถูกต้องชัดเจนและความเสี่ยงต่ำสุด ส่วน config ทั้งหมดใช้ค่าที่ระบบทำงานดีอยู่แล้ว → ทีมเดินหน้าเก็บข้อมูล/ทำ validation กับผู้เชี่ยวชาญได้เลย โดยเก็บการจูนละเอียดไว้เป็นการศึกษาในอนาคต
