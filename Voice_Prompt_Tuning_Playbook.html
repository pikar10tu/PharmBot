Playbook การจูน Prompt เสียง (Voice Prompt Tuning)

คู่มือทดสอบและปรับ system prompt ของผู้ป่วยจำลอง (Gemini Live) แบบมีตัวแปรควบคุม

🎯 เป้าหมาย: รองรับการซักประวัติทางเภสัช (WWHAM) 🧪 วิธี: controlled experiment 👥 สำหรับทีมพัฒนา

1 หลักการ (อ่านก่อนเริ่ม)

ทำไมต้องคุมตัวแปร — เพื่อให้รู้ว่า "prompt ที่เปลี่ยน" คือสาเหตุที่ทำให้ดีขึ้นจริง

บทบาทตัวแปรคืออะไรในการทดสอบนี้
ตัวแปรควบคุม (ล็อกไว้)บททดสอบเดียวกัน + เคสเดียวกัน + ค่า config เดียวกัน + คนทดสอบคนเดิม
ตัวแปรต้น (สิ่งที่เปลี่ยน)prompt เท่านั้น — เปลี่ยนทีละเวอร์ชัน
ตัวแปรตาม (สิ่งที่วัด)คุณภาพคำตอบของผู้ป่วยจำลอง ตาม rubric ในข้อ 4
กฎเหล็ก 2 ข้อ: (1) เปลี่ยน prompt อย่างเดียว ห้ามแตะ config อื่นพร้อมกัน · (2) รันแต่ละเวอร์ชัน 5 รอบ แล้วดูแนวโน้มรวม เพราะ AI ตอบไม่เหมือนกันทุกครั้ง (สุ่ม) — ห้ามตัดสินจากรอบเดียว

2 ล็อกค่าก่อนเริ่ม (กรอกครั้งเดียว ใช้ทุกเวอร์ชัน)

ทุกค่าด้านล่างต้องเท่ากันทุกการทดสอบ ถ้าจำเป็นต้องเปลี่ยน ให้เริ่มตารางผลใหม่

3 บททดสอบมาตรฐาน (คนทดสอบอ่านบท "เภสัชกร" เหมือนเดิมทุกรอบ)

บทเรียงตามลำดับการซักประวัติ WWHAM วัดการซักประวัติ สลับด้วย "กับดัก" ทดสอบ failure mode · สังเกตคำตอบของผู้ป่วย เทียบกับ "พฤติกรรมที่ควรเป็น"

#บทของเภสัชกร (อ่านออกเสียง)ทดสอบอะไร / พฤติกรรมที่ควรเป็น
1"สวัสดีครับ วันนี้มีอะไรให้ช่วยไหมครับ"เปิดบทสนทนา ตอบสั้นๆ แล้วบอกอาการหลัก ("เจ็บคอ") เท่านั้น ไม่เริ่มทักก่อน
2"มีอาการอะไรบ้างครับ"W-What (ปลายเปิด) ย้ำอาการหลัก ไม่เทข้อมูลระยะเวลา/แพ้ยา/โรคประจำตัวออกมาเอง
3"เป็นมากี่วันแล้วครับ"H-How long ตอบระยะเวลาตามเคส สมจริง สั้น
4"มีอาการอื่นร่วมด้วยไหม เช่น ไข้ ไอ น้ำมูก"อาการร่วม ตอบเฉพาะที่ถูกถาม ตามข้อมูลในเคส
5"ก่อนมาได้กินยาอะไรมาหรือยังครับ"A-Action taken ตอบการรักษา/ยาที่ใช้ก่อนหน้า
6"มีโรคประจำตัว หรือยาที่กินประจำไหมครับ"M-Medication เพิ่งเปิดเผยตรงนี้ (ต้องไม่เผยตั้งแต่ข้อ 2)
7"แพ้ยาอะไรไหมครับ"แพ้ยา (critical) ตอบชัด — หลังข้อนี้ควรเริ่ม signal ถามหายา (ดูข้อ 14)
8"ตอนนี้ตั้งครรภ์หรือให้นมบุตรอยู่ไหมครับ"ตั้งครรภ์ (critical, เคสหญิง) ตอบตามเคส
9"เมื่อเช้ารถติดมากไหมครับกว่าจะมาถึง"นอกบริบท งงๆ/ถามกลับ ไม่ออกนอกบทบาทผู้ป่วย
10"มีอาการ dysphagia ไหมครับ"ศัพท์แพทย์ ถามกลับว่า "คือยังไงหรอคะ" ไม่แกล้งเข้าใจ
11"Do you have a fever now?" (พูดอังกฤษ)ปักภาษา ต้องตอบ ภาษาไทย เสมอ ไม่สลับเป็นอังกฤษ
12"เคยผ่าตัดต่อมทอนซิลไหมครับ" (เคสไม่ได้ระบุ)ไม่แต่งข้อมูล ตอบ "ไม่แน่ใจ/จำไม่ได้" ไม่กุเรื่องใหม่
13เริ่มพูดแทรกขณะผู้ป่วยกำลังตอบ: "เดี๋ยวๆ ขอถามอีกเรื่อง..."พูดแทรก (barge-in) หยุดพูดทันที รอฟัง ไม่พูดต่อจากที่ค้าง
14(ไม่ต้องพูด — สังเกตเอง หลังข้อ 6–7)Signal รับยา ผู้ป่วยถาม "แล้วมียาอะไรให้กินได้บ้างคะ" ครั้งเดียว ไม่ถามซ้ำ

4 เกณฑ์ให้คะแนน (Rubric) — ให้คะแนน 1–5 ต่อข้อ

1 = แย่/ผิดชัดเจน · 3 = พอใช้ · 5 = ดีมาก/เป็นธรรมชาติ · ผู้ให้คะแนนควรไม่รู้ว่าเป็น prompt เวอร์ชันไหน (blind)

รหัสเกณฑ์
ด้านการซักประวัติ
H1คำถามปลายเปิด → ตอบแค่อาการหลัก ไม่เทข้อมูลทั้งหมด
H2ไม่เปิดเผยแพ้ยา/โรคประจำตัว จนกว่าถูกถามตรงๆ
H3ตอบแต่ละประเด็น (ระยะเวลา/อาการร่วม/ฯลฯ) ตรงและสมจริงตามเคส
H4Signal ถามหายาทำงานถูกจังหวะ และทำครั้งเดียว
H5ไม่แต่งข้อมูลที่ไม่มีในเคส (ตอบ "ไม่แน่ใจ/จำไม่ได้")
ด้านการพูด/เสียง
V1ตอบสั้น 1–2 ประโยค ไม่ยืดยาว
V2พูดภาษาไทยตลอด แม้เภสัชกรพูดอังกฤษ/ชื่อยาอังกฤษ
V3ไม่มีตัวเลข หัวข้อ หรือสัญลักษณ์ในคำพูด
V4คำลงท้าย (ค่ะ/ครับ) ไม่ถี่เกินคนจริง
V5อารมณ์/น้ำเสียงสมจริงตามสถานการณ์ ไม่บ่นทุกประโยค
V6หยุดทันทีเมื่อถูกพูดแทรก

5 ตารางบันทึกผล (ทำซ้ำต่อ 1 เวอร์ชัน prompt)

คัดลอกบล็อกนี้สำหรับ prompt แต่ละเวอร์ชัน · ใส่คะแนน 1–5 แต่ละรอบ แล้วหาค่าเฉลี่ย

Prompt เวอร์ชัน:  v....   เปลี่ยนอะไรจากเวอร์ชันก่อน:  

เกณฑ์รอบ1รอบ2รอบ3รอบ4รอบ5เฉลี่ย
H1 ปลายเปิด
H2 ไม่เผยก่อนถาม
H3 ตอบตรงสมจริง
H4 signal รับยา
H5 ไม่แต่งข้อมูล
V1 สั้น
V2 ภาษาไทยตลอด
V3 ไม่มีสัญลักษณ์
V4 คำลงท้ายพอดี
V5 อารมณ์สมจริง
V6 หยุดเมื่อแทรก
คะแนนรวมเฉลี่ย

บันทึกข้อสังเกต/ตัวอย่างคำตอบที่ดีหรือพัง:

6 ขั้นตอนการใช้งาน

  1. กรอกค่าล็อก (ข้อ 2) ให้ครบ — ใช้ค่าเดียวกันทุกเวอร์ชัน
  2. ตั้ง prompt เป็นเวอร์ชันที่จะทดสอบ (เปลี่ยนจากเวอร์ชันก่อนแค่จุดเดียวถ้าทำได้ จะรู้ว่าอะไรมีผล)
  3. คนทดสอบเข้าโหมดเสียง อ่านบทข้อ 3 ตามลำดับ ครบ 1 รอบ
  4. ผู้ให้คะแนน (คนละคนกับคนอ่านบทยิ่งดี) ให้คะแนนตาม rubric ข้อ 4 ลงตารางข้อ 5
  5. ทำซ้ำให้ครบ 5 รอบต่อเวอร์ชัน → หาค่าเฉลี่ย
  6. เปลี่ยน prompt เป็นเวอร์ชันถัดไป ทำซ้ำ แล้วเทียบค่าเฉลี่ยระหว่างเวอร์ชัน
  7. เลือกเวอร์ชันที่คะแนนรวมดีที่สุด + ดูว่าตกเกณฑ์ข้อไหนซ้ำๆ เพื่อแก้ prompt ต่อ
เคล็ดลับ: ถ้าเพิ่มกฎใน prompt แล้วคะแนน "ด้านการพูด" (V) ตกลง (เช่น พูดแข็ง) แสดงว่า over-instruction → ลองตัดกฎที่ไม่จำเป็นออก กฎทุกข้อต้องคุ้มค่าที่จะอยู่
